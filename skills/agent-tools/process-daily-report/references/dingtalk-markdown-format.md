# 钉钉 Markdown 格式

将 LLM 返回的 JSON 转换为适合钉钉 Markdown 的高亮格式。优先使用 HTML 的 `<font color="...">...</font>` 做文字着色，同时保留 `**加粗**`、emoji 和反引号作为降级兜底。

## 颜色语义

| 类型 | 颜色 | 推荐色值 | 突出词示例 |
|------|------|----------|------------|
| 进展 | 绿色 | `#16A34A` | `已闭环`、`已打通`、`对齐`、`成功` |
| 风险 | 橙色 | `#F59E0B` | `待定位`、`流同步超时` |
| 高风险 | 红色 | `#DC2626` | `阻塞主线`、阻塞项 |
| Action | 蓝色 | `#2563EB` | 日期、负责人、`定位`、`验证`、`对齐` |

颜色标签尽量包裹标题、风险标签、结果词、动作词等关键内容，不必给整段全文上色。

## 模板示例

```markdown
# 日报摘要 - {日期}

## 🟢 <font color="#16A34A">**进展**</font>
- ✅ <font color="#16A34A">**进展项1**</font>，关键结果：<font color="#16A34A">`已闭环`</font>
- ✅ <font color="#16A34A">**进展项2**</font>，关键结果：<font color="#16A34A">`已打通`</font>

## 🟠 <font color="#F59E0B">**风险**</font>
- ⚠️ <font color="#F59E0B">`流同步超时`</font>：风险项1 | 应对措施：优化同步机制 | 责任人：张三 | 完成时间：03/25
- ⚠️ <font color="#F59E0B">`待定位`</font>：风险项2 | 应对措施：增加日志排查 | 责任人：李四 | 完成时间：03/26

## 🔴 <font color="#DC2626">**高风险**</font>
- 🚨 <font color="#DC2626">`阻塞主线`</font>：风险项3 | 应对措施：上升求助HW内部研发 | 责任人：王五 | 完成时间：03/27

## 🔵 <font color="#2563EB">**Action**</font>
- 📌 <font color="#2563EB">`03/22`</font> <font color="#2563EB">**Action项1**</font>，负责人：<font color="#2563EB">`张三`</font>
- 📌 <font color="#2563EB">`03/23`</font> <font color="#2563EB">**Action项2**</font>，关键动作：<font color="#2563EB">`验证`</font>
```

## 显示规则

- 如果某类为空，则不显示该部分
- 只有存在高风险项时才显示 `## 🔴 **高风险**` 部分
- 如果客户端没有正确渲染颜色，保留的 emoji、加粗、反引号仍然需要保证可读性
- 风险行格式：`风险描述 | 应对措施：xxx | 责任人：xxx | 完成时间：xxx`

## curl 转义注意

在 shell 中构建 `MARKDOWN_TEXT` 时，JSON 内的双引号需要转义为 `\"`，反引号按需转义为 `` \` ``。
