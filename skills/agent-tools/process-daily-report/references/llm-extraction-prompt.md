# LLM 提取 Prompt

将以下 prompt 发送给 LLM，将 `{content}` 替换为日报正文：

```text
请分析以下日报内容，提取并返回结构化的 JSON（只返回 JSON，不要其他内容）：

{
  "进展": ["关键进展项1", "关键进展项2", ...],
  "风险": [
    {
      "description": "风险项描述",
      "severity": "medium 或 high",
      "mitigation": "应对措施",
      "owner": "责任人",
      "deadline": "完成时间"
    },
    ...
  ],
  "Action": ["Action项1（含负责人+日期）", "Action项2", ...]
}

规则：
- 只提取最重要的信息，每类不超过 5 条
- 进展：已完成、已通过、已对齐、正常等正向内容
- 风险：问题、待定位、报错、超时、待确认等需关注内容
- **每个风险必须包含应对措施（mitigation）、责任人（owner）、完成时间（deadline）**
- Action：下一步计划、负责人+日期的任务项
- 为每条内容补充一个 `highlight` 字段，用于标记最值得强调的词或短语
- 为风险补充 `severity` 字段：`medium` 表示一般风险，`high` 表示高风险或阻塞项
- 如果某类为空数组，则不返回该字段
- **重要**：如果风险项已经在"风险"部分列出了应对措施，则不要在"Action"部分重复列出

日报内容：
{content}
```

获取 LLM 返回的 JSON 后解析。如果 JSON 无效，提示用户重试或手动修正提取结果。
